# Sesión 12: Testing de Contratos Inteligentes con Foundry

Para avanzar en nuestro camino como desarrolladores de contratos inteligentes, es importante dar un salto al uso de Frameworks, que nos ayude a tener más herramientas a la mano y actuar con mayor detalle y libertad.

Hoy en día el framework más popular para el desarrollo de contratos inteligente EVM es Foundry. Se caracteriza por ser brindar una experiencia de desarrollo rápida y modular, diseñado para compilar, testear, depurar y desplegar contratos inteligentes, todo usando solo el lenguaje de Solidity.

Herramientas que veremos:

* **forge** → compilar, testear, scripts de deploy, coverage, etc.
* **cast** → CLI para interactuar con RPCs y contratos (call, send, decode, etc.)
* **anvil** → nodo local EVM, rápido y útil para pruebas
* **chisel** → REPL para probar snippets de Solidity rápido
